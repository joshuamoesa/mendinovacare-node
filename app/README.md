## Tech Stack

- **Express** — HTTP server
- **EJS + express-ejs-layouts** — server-side templates with shared layout
- **Prisma** — database ORM (SQLite by default, swap `DATABASE_URL` for Postgres)
- **TypeScript** — compiled via `ts-node` in dev

## How to Run

```bash
# 1. Install dependencies
npm install

# 2. Configure environment
cp .env.example .env
# Edit .env — set DATABASE_URL (default: file:./dev.db for SQLite)

# 3. Push schema and generate Prisma client
npm run db:push

# 4. Start development server
npm run dev
```

Open [http://localhost:3001](http://localhost:3001)

## What Has Been Fixed (post-generation improvements)

The generator has been tuned based on the Mendinova Care model. These fixes are baked
into the generators and will persist on future regenerations:

### Hero / background-image pages
- **Duplicate hero removed** — Mendix stores both a mobile and desktop variant of the
  hero widget with the same image. The generator now keeps only the first occurrence.
- **No dark overlay** — the background image is light-toned; text uses dark navy/orange.
- **Background positioned `right center`** — text sits on the white left half of the image.
- **Hero headline styles** — first `<p>` in `.mx-col-12` → 3.5rem navy; second → 2.5rem orange.
- **Heading promotion suppressed** on hero pages — the hero itself provides the visual hierarchy.

### DataView names
- Internal Mendix widget names (`dataView5`, `dataView3`, …) are suppressed as headings.
- DataViews with no editable children render as `<div>` wrappers, not `<form>` elements.

### Layout grid
- LayoutGrid rows and columns are preserved as flex rows (`.mx-row` / `.mx-col-N`)
  instead of being flattened into a single children list.
- Auto-width columns (Mendix weight = -1) get class `mx-col--1`.

### System entities
- System entities (e.g. `UserRole`) are not in the Prisma schema.
- Pages whose primary entity is a system entity generate an empty list
  (`const userroleList: unknown[] = []`) rather than calling a non-existent Prisma model.

## Known Limitations

- **Conditional visibility** — Mendix widget show/hide conditions are not rendered;
  all widgets display unconditionally.
- **Custom widgets** — Atlas carousel, icon, progress, chart widgets render as
  `<!-- CustomWidget -->` placeholder comments.
- **Section heading styles** — paragraph labels like "Why choose our portal?" and
  "24/7 Access" are not yet mapped to semantic heading elements because Atlas
  font/heading design properties are not captured from the Mendix model.
- **Microflows** — complex microflow logic is stubbed; only the call signature is generated.
